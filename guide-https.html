HTTPS on Cloudflare

Enabling SSL so every service on dustyserver.co.in is served over HTTPS — free, automatic, zero certificate management on spserver. Visitors see the padlock, data is encrypted, and the home IP stays hidden.

01 — Concept
What is HTTPS

HTTPS stands for HTTP Secure. It is the same HTTP protocol used for all web communication, but with an encryption layer added on top called TLS (Transport Layer Security). When a browser connects to a site over HTTPS, all data exchanged — the page content, any form inputs, cookies, credentials — is encrypted end to end.

The padlock icon in the browser address bar means the connection is HTTPS. Without it, anyone on the same network (your ISP, a cafe WiFi) can see exactly what you send and receive. With HTTPS, they see only encrypted noise.

HTTP — no encryption
All data transmitted in plain text. Anyone with network access (ISP, router, middleman) can read your login credentials, session cookies, and all page content. Browser shows "Not Secure" warning.
HTTPS — encrypted
All data encrypted with TLS. Even if intercepted, it's unreadable without the private key. Browser shows the padlock. Required for any modern web service — browsers distrust plain HTTP.
ℹ HTTPS is no longer optional. Modern browsers show "Not Secure" warnings for plain HTTP sites. Google's search ranking penalises non-HTTPS sites. Password managers and payment forms refuse to work without it. Every public service needs HTTPS.
02 — Protocol
TLS / SSL — what's the difference

SSL (Secure Sockets Layer) was the original encryption protocol, developed in the 1990s. It was replaced by TLS (Transport Layer Security) in 1999. SSL 2.0 and 3.0 are both deprecated and insecure. Despite this, the term "SSL" stuck — people still say "SSL certificate" when they mean a TLS certificate, and Cloudflare's settings section is called "SSL/TLS".

In practice: when someone says SSL they mean TLS. The current standard is TLS 1.3 (released 2018), with TLS 1.2 still widely supported. TLS 1.0 and 1.1 are deprecated. Cloudflare enforces TLS 1.2 minimum by default.

Protocol timeline: SSL 2.0 (1995) ← broken, never use SSL 3.0 (1996) ← broken (POODLE attack), never use TLS 1.0 (1999) ← deprecated 2021 TLS 1.1 (2006) ← deprecated 2021 TLS 1.2 (2008) ← still widely used, acceptable TLS 1.3 (2018) ← current standard, fastest and most secure Cloudflare uses TLS 1.3 for all connections on our domain ✓
03 — Architecture
How Cloudflare handles SSL for spserver

Since Cloudflare sits between the visitor and our server (via Cloudflare Tunnel), SSL is handled at two separate legs of the connection. Cloudflare issues and manages the certificate on its edge — our server only ever receives plain HTTP from the tunnel.

1
Visitor browser → Cloudflare edge
Encrypted HTTPS connection. Cloudflare presents its SSL certificate for *.dustyserver.co.in. Browser verifies, sees padlock ✓
2
Cloudflare edge → Cloudflare Tunnel → spserver
Plain HTTP over the already-encrypted Cloudflare Tunnel connection. spserver receives plain HTTP on localhost:80 — no SSL certificate needed on the server.
3
spserver → Nginx → response
Nginx serves files or proxies to apps. Response goes back through the same path. Visitor receives encrypted HTTPS response.
✓ The Cloudflare Tunnel connection itself is already encrypted with TLS — so even in Flexible mode, the connection between Cloudflare and spserver is not transmitted in plain text over the internet. It goes through an encrypted tunnel.
04 — SSL modes
Cloudflare SSL/TLS modes compared
Off
No encryption
HTTP only. Visitors get no padlock. Browser shows "Not Secure". Never use for any real site.
Flexible our choice
HTTPS visitor ↔ Cloudflare / HTTP Cloudflare ↔ server
Visitors get HTTPS and the padlock. No SSL certificate needed on spserver. The Cloudflare Tunnel itself is already encrypted. Perfect for our setup.
Full
HTTPS both legs, self-signed cert OK
HTTPS visitor ↔ Cloudflare AND HTTPS Cloudflare ↔ server. Server needs an SSL cert but it can be self-signed. More secure than Flexible.
Full (Strict)
HTTPS both legs, valid cert required
Same as Full but server must have a certificate from a trusted CA. Most secure — requires installing Cloudflare Origin Certificate on Nginx.
ℹ For a future upgrade to Full Strict: Cloudflare offers free Origin Certificates valid for 15 years — generated in the Cloudflare dashboard and installed on Nginx. This is the recommended production setup when you want end-to-end TLS verification.
05 — Decision
Why Flexible mode is correct for our setup

Flexible mode is sometimes criticised for not encrypting the connection between Cloudflare and the origin server. In a typical setup (Cloudflare → internet → your server), this means the last mile is unencrypted. However, our setup is different.

Traffic from Cloudflare to spserver does not travel over the open internet — it goes through a Cloudflare Tunnel, which is an encrypted TLS connection maintained by cloudflared. Even in Flexible mode, the Cloudflare-to-server leg is encrypted. The "Flexible" label applies to whether Cloudflare verifies a certificate on our server — not to whether the connection is encrypted.

Visitor ────── HTTPS (TLS 1.3) ────── Cloudflare edge ↓ Cloudflare edge ──── Encrypted Tunnel (TLS) ──── cloudflared on spserver ↓ cloudflared ──── plain HTTP internally ──── Nginx on localhost:80 Both legs are encrypted in practice ✓ The "Flexible" label only means Cloudflare doesn't verify our origin cert

Step 1 — Set SSL mode to Flexible

Configure SSL/TLS encryption mode in Cloudflare dashboard

# 2. Select dustyserver.co.in

✓ Once set to Flexible, all subdomains on dustyserver.co.in automatically get HTTPS — dustyserver.co.in, terminal.dustyserver.co.in, flask.dustyserver.co.in, all of them. No per-subdomain configuration needed.

Step 2 — Always Use HTTPS
2
Enable automatic HTTP → HTTPS redirect
▼

Even with SSL enabled, someone could still type http://dustyserver.co.in and get a plain HTTP response. "Always Use HTTPS" tells Cloudflare to automatically redirect any HTTP request to HTTPS — the visitor always ends up on the secure version regardless of what they typed.

Cloudflare dashboard
# 1. SSL/TLS → Edge Certificates
# 2. Find "Always Use HTTPS"
# 3. Toggle ON
#
# Result: http://dustyserver.co.in
#    automatically becomes https://dustyserver.co.in
# Status code: 301 Moved Permanently (permanent redirect)
03 — HSTS (optional)
Step 3 — HTTP Strict Transport Security
3
Enable HSTS — tell browsers to always use HTTPS
▼

HSTS is a security header that tells browsers "this site only works over HTTPS — never try HTTP". The browser remembers this and refuses to make plain HTTP connections to your domain for the specified duration, even if explicitly typed. This prevents certain downgrade attacks. Warning: once enabled with a long max-age, you cannot easily go back to HTTP. Only enable if you're committed to HTTPS permanently.

Cloudflare dashboard
# 1. SSL/TLS → Edge Certificates
# 2. Find "HTTP Strict Transport Security (HSTS)"
# 3. Click "Enable HSTS"
# 4. Recommended settings:
#    Max Age Header: 6 months (15768000 seconds)
#    Include subdomains: ON
#    Preload: OFF (leave off unless you know what you're doing)
⚠ HSTS with a long max-age is a one-way door. If you ever disable HTTPS or let the domain lapse, visitors who have the HSTS header cached will get errors for months — their browser will refuse to connect over HTTP. For a personal lab server this is fine, but understand the commitment.

01 — Certificates
What is an SSL certificate

An SSL certificate is a digital document that serves two purposes: it proves that a public key belongs to a specific domain, and it contains the public key itself that browsers use to establish the encrypted connection. Certificates are issued by Certificate Authorities (CAs) — trusted organisations that browsers recognise.

When your browser connects to dustyserver.co.in, Cloudflare presents its certificate. The browser checks: (1) is the certificate for the right domain? (2) is it issued by a CA the browser trusts? (3) is it still valid (not expired)? If all three pass — the padlock appears and the connection is established.

Certificate contains: Domain: *.dustyserver.co.in (wildcard — covers all subdomains) Issued by: Google Trust Services (Cloudflare's CA partner) Valid from: date of issuance Valid until:~90 days later (Cloudflare auto-renews before expiry) Public key: used to establish encrypted session Cloudflare manages this certificate automatically No manual renewal, no Let's Encrypt setup, no cronjobs needed
02 — TLS handshake
What happens when a browser connects over HTTPS

Before any HTTP data is exchanged, the browser and server perform a TLS handshake — a negotiation that establishes the encryption parameters and authenticates the server. In TLS 1.3 this takes just one round trip (compared to two in TLS 1.2), making HTTPS nearly as fast as plain HTTP.

1
Client Hello
Browser sends: supported TLS versions, cipher suites, random number, and key share data.
2
Server Hello + Certificate
Cloudflare replies with: chosen cipher suite, its certificate (containing its public key), and its own key share data. In TLS 1.3 the server also sends encrypted data already.
3
Certificate verification
Browser verifies the certificate — checks domain matches, CA is trusted, cert not expired. Both sides derive the same session key from the key share data (Diffie-Hellman).
4
Encrypted HTTP begins
Both sides now have a shared session key. All subsequent HTTP data is encrypted. The GET request, the HTML response, cookies — all encrypted. Handshake complete.

01 — Issues and fixes
Common HTTPS problems
Browser shows "Not Secure" or ERR_SSL_PROTOCOL_ERROR
SSL mode may be set to Off in Cloudflare, or Cloudflare is not proxying the domain. Check: (1) Cloudflare dashboard → SSL/TLS → Overview — is a mode selected? (2) Cloudflare DNS tab — is the orange cloud (proxy) enabled for your domain's DNS records? If the cloud is grey (DNS only), Cloudflare is not proxying traffic and cannot provide SSL. Click the cloud to enable proxying.
ERR_TOO_MANY_REDIRECTS — infinite redirect loop
Classic sign that SSL mode is set to Flexible but Nginx on spserver is also redirecting HTTP to HTTPS. The result: visitor → Cloudflare (HTTPS) → Nginx (HTTP, redirects to HTTPS) → Cloudflare (HTTPS) → loop. Fix: remove any HTTP-to-HTTPS redirect rules from Nginx config — let Cloudflare handle it via "Always Use HTTPS". Or change SSL mode to Full if Nginx has a certificate.
SSL certificate error — certificate not for this domain
Browser sees a certificate mismatch. This usually happens if you are bypassing Cloudflare and connecting directly to spserver (which has no SSL cert). In normal operation through Cloudflare Tunnel, visitors never connect directly to spserver — they always go through Cloudflare, which presents the correct certificate. If you see this, check if someone has the server's direct IP and is connecting without Cloudflare.
HTTPS works for main domain but not subdomains
Cloudflare's Universal SSL covers *.dustyserver.co.in (one level of wildcards). If a subdomain is not proxied through Cloudflare (grey cloud in DNS tab), it won't have SSL. Also check: is the subdomain's CNAME record in Cloudflare DNS? If the record doesn't exist, the subdomain won't work at all. Check Cloudflare DNS tab for all subdomain records.
How to upgrade from Flexible to Full Strict
Generate a Cloudflare Origin Certificate: Cloudflare dashboard → SSL/TLS → Origin Server → Create Certificate. Download the certificate and key. On spserver, save them to /etc/ssl/certs/ and configure Nginx to use them with listen 443 ssl. Then change Cloudflare SSL mode to Full Strict. This gives genuine end-to-end TLS verification.
01 — What's next
Future scope
⟶ Things to explore from here
Upgrade to Full Strict mode — generate a Cloudflare Origin Certificate and install it on Nginx. Change SSL mode to Full Strict. Gives genuine TLS verification on both legs of the connection. More secure, minimal extra effort.
Cloudflare Minimum TLS Version — in SSL/TLS → Edge Certificates, set minimum TLS version to 1.2 or 1.3. Prevents connections from very old clients using deprecated TLS versions. Already at 1.2 by default — consider bumping to 1.3 only.
Certificate Transparency monitoring — Cloudflare can notify you if a certificate is ever issued for your domain outside of Cloudflare. Detects if someone obtained a fraudulent certificate for dustyserver.co.in. Enable in Cloudflare dashboard → SSL/TLS → Edge Certificates → Certificate Transparency Monitoring.
Let's Encrypt on Nginx directly — if you ever want to serve content directly without Cloudflare Tunnel (e.g. from a VPS with a real IP), Let's Encrypt with certbot provides free SSL. sudo apt install certbot python3-certbot-nginx then sudo certbot --nginx. Auto-renews via cron.
HSTS preloading — submit dustyserver.co.in to the HSTS preload list (hstspreload.org). Browsers ship with a built-in list of domains that must use HTTPS — your domain gets baked into Chrome, Firefox, Safari. The strongest possible HTTPS enforcement. Irreversible — only do this if permanently committed to HTTPS.
