Nginx Docker Reverse Proxy for Containers

Phase 2 — Completed

Overview

We moved Nginx inside Docker so it becomes the single gateway for all containerised apps. Flask now runs internally only on the shared Docker network appnet. Cloudflare Tunnel points to Nginx (port 8082), which forwards traffic to Flask.

Concepts

  • Docker Network (appnet) — isolated network so containers can reach each other by service name (http://flask:5000)
  • Reverse Proxy — Nginx receives public traffic and forwards it internally to Flask
  • Internal-only ports — Flask port 5000 is no longer exposed to the host
  • Port 8082 — only Nginx container publishes to the host

Final Config Files

1. Flask docker-compose.yml (updated)

services:
  flask:
    build: .
    expose:
      - "5000"
    volumes:
      - .:/app
    restart: always
    networks:
      - appnet

networks:
  appnet:
    external: true

2. Nginx docker-compose.yml

version: '3.8'

services:
  nginx:
    image: nginx:alpine
    container_name: nginx-docker-proxy
    restart: unless-stopped
    ports:
      - "8082:80"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/conf.d/default.conf:ro
    networks:
      - appnet

networks:
  appnet:
    external: true

3. nginx/nginx.conf

server {
    listen 80;
    server_name flask.dustyserver.co.in;

    location / {
        proxy_pass http://flask:5000;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_redirect off;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
    }
}

Blockers Hit & How We Solved Them

  • Old nginx_app folder leftover → completely removed
  • appnet network not found → created with docker network create appnet
  • Typo in external:true → fixed spacing
  • Flask still bound to host port 5000 → removed ports, added expose + network

Troubleshoot

  • 502 Bad Gateway → service name mismatch in proxy_pass (must be flask)
  • Connection refused → both containers must be on appnet
  • Port 8082 in use → sudo ss -tlnp | grep 8082

Future Scope

  • Move traditional host Nginx traffic to Docker Nginx too
  • Add more services with their own subdomains
