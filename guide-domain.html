Domain + DNS

Registering dustyserver.co.in, moving DNS management to Cloudflare, and understanding how the internet translates a domain name into a connection to spserver.

01 — Concept
What is DNS

DNS stands for Domain Name System. It is the internet's phonebook — it translates human-readable domain names like dustyserver.co.in into IP addresses that computers use to connect. Without DNS, you would have to remember and type IP addresses for every website you visit.

A domain name is structured hierarchically from right to left. In flask.dustyserver.co.in: .in is the top-level domain (country code for India), .co is a second-level domain under .in (short for commercial), dustyserver is the registered name, and flask is a subdomain.

flask . dustyserver . co . in ↑ ↑ ↑ ↑ subdomain registered second top-level (you create) name level domain (TLD) (you buy)
ℹ DNS is a distributed system — not one server knows everything. Thousands of servers around the world hold pieces of the DNS database and query each other as needed. This is why DNS changes can take minutes to hours to propagate globally.
02 — How it works
The full DNS lookup flow

When you type dustyserver.co.in in a browser, a multi-step lookup happens in milliseconds:

Browser types: dustyserver.co.in ↓ 1. Check browser DNS cache → not found 2. Check OS DNS cache → not found 3. Ask configured DNS resolver (8.8.8.8 — Google DNS) ↓ 4. Resolver asks root nameserver: "who handles .in?" Root server replies: "ask ns1.registry.in" ↓ 5. Resolver asks .in nameserver: "who handles dustyserver.co.in?" .in server replies: "ask kobe.ns.cloudflare.com" ↓ 6. Resolver asks Cloudflare nameserver: "what is dustyserver.co.in?" Cloudflare replies: "104.21.33.66" (Cloudflare edge IP) ↓ 7. Browser connects to 104.21.33.66 8. Cloudflare routes through tunnel to spserver Website loads ✓

Notice that step 5 is where our nameserver setting matters. By changing GoDaddy's nameservers to point to Cloudflare (kobe.ns.cloudflare.com), we made Cloudflare the authority for all DNS records under dustyserver.co.in. Cloudflare now answers all DNS queries for our domain.

03 — Record types
DNS record types
A
Address record
Maps a domain to an IPv4 address. The most fundamental record. We don't need one — Cloudflare Tunnel creates CNAME records instead.
CNAME
Canonical Name
Maps a domain to another domain name (alias). All our subdomains are CNAMEs pointing to Cloudflare's tunnel endpoint. Created automatically by cloudflared tunnel route dns.
NS
Nameserver
Defines which servers are authoritative for this domain. We have two: kobe.ns.cloudflare.com and saanvi.ns.cloudflare.com. Cannot be deleted — required.
SOA
Start of Authority
Master record for the zone — contains admin info. Auto-managed by Cloudflare. Cannot be deleted.
AAAA
IPv6 Address
Maps a domain to an IPv6 address. Equivalent of A record for IPv6. Not configured on dustyserver.co.in yet — future improvement.
MX
Mail Exchange
Points to mail servers for the domain. Needed if you want @dustyserver.co.in email addresses. Not set up yet.
TXT
Text record
Stores arbitrary text. Used for domain verification (Google Search Console, email SPF/DKIM), and other services that need to confirm you own the domain.
SRV
Service record
Specifies the host and port for a specific service. Used for VoIP, game servers, and other protocols. Not needed for web hosting.
04 — Nameservers
What are nameservers and why they matter

Nameservers are the servers that answer DNS queries for your domain. When you register a domain, the registrar (GoDaddy in our case) assigns their own nameservers by default. Whoever controls the nameservers controls all DNS records for the domain.

By changing the nameservers to Cloudflare's, we handed DNS management to Cloudflare. GoDaddy still owns the registration — they hold the contract with the .in registry. But Cloudflare now manages all DNS records, answers all DNS queries, and controls what IP addresses the domain resolves to.

GoDaddy's role
Domain registrar — holds the legal registration contract with the .in registry. Pays the yearly renewal fee. That's it. GoDaddy has no control over DNS records anymore.
Cloudflare's role
DNS manager — answers all queries for dustyserver.co.in. Controls all A, CNAME, MX, TXT records. Also provides SSL certificates, DDoS protection, Cloudflare Tunnel routing, and analytics.
ℹ Our Cloudflare nameservers: kobe.ns.cloudflare.com and saanvi.ns.cloudflare.com. These are assigned uniquely per domain — yours may be different nameserver names in the kobe/saanvi family.

Method 1 — CLI (recommended)
Use the cloudflared command. Creates the CNAME automatically. Faster and less error-prone than dashboard.

Method 2 — Cloudflare Dashboard
Go to DNS → Add record → Type: CNAME → Name: newapp → Target: b72cf76b-0820-44cb-90df-f4c58ed87184.cfargotunnel.com → Save.
⚠ Only use second-level subdomains — e.g. newapp.dustyserver.co.in. Third-level subdomains like new.app.dustyserver.co.in cause DNS issues on Cloudflare's free plan. This was learned the hard way during the Flask setup.
03 — Propagation
DNS propagation explained

After creating or changing a DNS record, the change must spread to DNS resolvers worldwide. This is called DNS propagation. DNS resolvers cache records for a time period defined by the record's TTL (Time To Live). Until the cache expires, old resolvers still return old values.

Cloudflare sets a TTL of 1 minute for records managed through Cloudflare Tunnel (CNAMEs). This means new subdomains propagate very quickly — typically 1-5 minutes for most of the world. Some resolvers may take longer if they override TTL with a minimum cache time.

You run: cloudflared tunnel route dns spserver newapp.dustyserver.co.in Cloudflare creates CNAME record immediately DNS propagation timeline: 1-2 min: Cloudflare's own resolvers know (1.1.1.1 etc.) 2-5 min: Most major resolvers updated (Google 8.8.8.8, etc.) 5-30 min: Regional and ISP resolvers updated Up to 24h: Rare edge cases — resolvers ignoring TTL Test if ready: nslookup newapp.dustyserver.co.in Ready when it returns Cloudflare IPs (104.21.x.x or 172.67.x.x)
ℹ If a subdomain works from your phone on mobile data but not from your PC at home — your home router or ISP's DNS is caching the old value. Try incognito mode (browser cache bypass) or flush your PC's DNS: ipconfig /flushdns on Windows.
01 — DNS diagnostics
DNS checking and diagnostic commands
nslookup dustyserver.co.in
Basic DNS lookup — shows IP address the domain resolves to. Should return Cloudflare IPs.
nslookup -type=NS dustyserver.co.in
Look up nameservers for the domain — should return kobe.ns.cloudflare.com and saanvi.ns.cloudflare.com after propagation.
nslookup -type=CNAME flask.dustyserver.co.in
Look up CNAME record for a specific subdomain — shows the tunnel endpoint it points to.
nslookup -type=MX dustyserver.co.in
Look up mail exchange records — useful when setting up email in the future.
dig dustyserver.co.in
Detailed DNS query output — shows full response including TTL, query time, and which server answered. Install: sudo apt install dnsutils
dig dustyserver.co.in +short
Dig with short output — just the IP addresses, nothing else.
dig NS dustyserver.co.in
Query nameserver records with dig — more detail than nslookup.
dig @8.8.8.8 dustyserver.co.in
Query specifically through Google DNS — bypasses your local resolver cache to get a fresh answer.
dig @1.1.1.1 dustyserver.co.in
Query through Cloudflare DNS — shows what Cloudflare returns for the domain.
dig +trace dustyserver.co.in
Trace the full DNS resolution path — shows each step from root servers to Cloudflare nameservers. Educational and useful for debugging.
ipconfig /flushdns
Windows — flush the local DNS cache. Run when a subdomain works on mobile but not on your PC.
sudo resolvectl flush-caches
Linux — flush systemd-resolved DNS cache on spserver.
01 — What went wrong
Blockers hit during domain setup
⚠ Blocker 1
Domain showed "On Hold" after purchase — unusable
Immediately after purchasing dustyserver.co.in on GoDaddy, the domain showed status "On Hold" in the GoDaddy dashboard. Attempts to add it to Cloudflare failed or showed it as unverified.
✓ Fix
Check email inbox — click GoDaddy's verification link
GoDaddy sent a verification email to the registered email address immediately after purchase. ICANN requires all new domain registrations to verify the registrant's email within 15 days — if not verified, the domain is suspended. Opening the email and clicking "Verify" changed the domain status from "On Hold" to "Active" within minutes.
⚠ Blocker 2
Nameserver change in GoDaddy not taking effect
After updating nameservers in GoDaddy to Cloudflare's, running nslookup -type=NS dustyserver.co.in still returned GoDaddy's nameservers after 30 minutes. Cloudflare dashboard still showed "Pending" status.
✓ Resolution
Normal propagation delay — wait and verify again
This was normal DNS propagation time — not an error. The .in TLD registry takes time to update their records after a nameserver change is submitted. After waiting ~2 hours and running nslookup again, Cloudflare's nameservers were returned. Cloudflare dashboard also changed from "Pending" to "Active". No action was required — just patience.
⚠ Blocker 3
Third-level subdomain (flask.test.dustyserver.co.in) caused DNS issues
During Flask app setup, used flask.test.dustyserver.co.in as the subdomain. The cloudflared tunnel route dns command ran without error, but the subdomain either didn't resolve or showed SSL certificate errors.
✓ Fix
Switch to second-level subdomain — flask.dustyserver.co.in
Cloudflare's free plan SSL certificates cover *.dustyserver.co.in (one level of wildcard) but not *.*.dustyserver.co.in (two levels). A third-level subdomain like flask.test.dustyserver.co.in is not covered by the wildcard certificate. The fix was to use flask.dustyserver.co.in directly — a single-level subdomain covered by Cloudflare's certificate. Works perfectly since.
01 — Issues and fixes
Common DNS and domain problems
Domain not resolving — nslookup returns nothing or NXDOMAIN
Causes and fixes: (1) Nameservers not propagated yet — wait and check again with nslookup -type=NS dustyserver.co.in. (2) DNS record not created — check Cloudflare dashboard DNS tab, create the missing record. (3) Domain "On Hold" in GoDaddy — verify email. (4) Domain expired — renew in GoDaddy dashboard. (5) Wrong DNS record type — ensure CNAME points to the correct tunnel endpoint.
SSL certificate error — "not secure" warning in browser
Possible causes: (1) Third-level subdomain not covered by wildcard certificate — switch to second-level subdomain. (2) Cloudflare SSL mode set to "Off" — check SSL/TLS settings in dashboard. (3) Tunnel is bypassed and browser is connecting directly to Nginx on HTTP — check Cloudflare proxy status (orange cloud icon in DNS tab must be enabled).
Subdomain works on mobile data but not on home PC
Your home router or ISP's DNS server is caching the old DNS record. Fix: (1) Try incognito window — bypasses browser cache but not OS/router cache. (2) Flush Windows DNS: ipconfig /flushdns in Command Prompt. (3) Restart router to clear its DNS cache. (4) Change your PC's DNS to 8.8.8.8 temporarily to bypass ISP's slow-updating resolver.
Cloudflare shows domain as "Pending" for more than 24 hours
In rare cases, the .in TLD registry is slow to propagate nameserver changes. Check: (1) Verify you saved the correct Cloudflare nameservers in GoDaddy — copy-paste from Cloudflare dashboard, don't type from memory. (2) Run dig +trace dustyserver.co.in to see which nameservers are actually responding. (3) Contact GoDaddy support if it has been more than 48 hours — may need manual intervention.
NS and SOA records in Cloudflare can't be deleted
This is expected and correct. NS records define the authoritative nameservers — they are required for the domain to work. SOA is the zone's master record — also required. Cloudflare protects these records and won't let you delete them. They are auto-managed by Cloudflare. No action needed.
How to check if a record propagated globally
Use dnschecker.org or whatsmydns.net — these tools check your domain from multiple locations around the world simultaneously and show where propagation has completed. Useful when you want to know if a change has reached specific regions (India, US, Europe etc.).
